## 开发者的困境

作为开发者，我们经常需要安装各种开发工具来提高工作效率。然而，这些工具往往会：
- 在系统各处留下配置文件
- 修改环境变量
- 创建缓存和数据目录
- 难以完全卸载干净

特别是像 **Claude Code CLI** 这样的新兴 AI 开发工具，它们的安装方式可能不够标准化，卸载时更是让人头疼。

## 解决方案：开发工具一键卸载器

为了解决这个问题，我开发了一个专门针对开发工具的**一键卸载工具**。这个工具能够：

### 🎯 全面扫描
- **可执行文件**：在 `/usr/local/bin`、`~/.local/bin`、`/opt/homebrew/bin` 等常见位置查找
- **配置文件**：在 `~/.config`、`~/Library/Application Support`、`~/.cache` 等目录中搜索
- **环境变量**：检测与工具相关的环境变量设置
- **包管理器**：自动识别 Homebrew、npm、pip、Cargo 等包管理器中的安装记录

### 🔒 安全操作
- **预览模式**：先显示会删除什么，让你确认后再执行
- **逐个确认**：每个文件都会询问是否删除，避免误删重要数据
- **跨平台支持**：同时支持 macOS 和 Linux 系统

### 🚀 智能识别
工具内置了常见开发工具的识别规则，包括：
- Claude 相关工具
- Node.js/npm 工具
- Python/pip 工具  
- Rust/Cargo 工具
- Go 工具
- 以及其他各种 CLI 工具

## 使用方法

### 1. 下载并赋予执行权限
```bash
# 下载脚本（假设从 GitHub 下载）
chmod +x dev-uninstaller-enhanced.sh
```

### 2. 预览模式（安全第一）
```bash
# 查看 Claude 相关的所有文件，但不删除
./dev-uninstaller-enhanced.sh --dry-run claude
```

### 3. 正式清理
```bash
# 确认无误后，正式清理所有 Claude 相关文件
./dev-uninstaller-enhanced.sh claude
```

### 4. 高级选项
```bash
# 强制模式（跳过确认，谨慎使用）
./dev-uninstaller-enhanced.sh --force claude

# 详细输出模式
./dev-uninstaller-enhanced.sh --verbose --dry-run node
```

## 技术实现亮点

### 多层搜索策略
工具采用了多层搜索策略，确保不遗漏任何文件：
1. **文件系统扫描**：递归搜索常见目录
2. **包管理器查询**：调用各包管理器的列表命令
3. **环境变量分析**：解析当前 shell 环境
4. **智能模式匹配**：使用通配符匹配工具名称变体

### 安全保障机制
- 所有删除操作都需要用户确认（除非使用 `--force`）
- 支持完整的 dry-run 模式进行预览
- 错误处理和异常恢复机制
- 详细的日志记录

### 可扩展架构
通过 JSON 配置文件定义新的工具识别规则，轻松扩展支持更多开发工具。
