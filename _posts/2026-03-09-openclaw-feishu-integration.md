# OpenClaw + 飞书集成配置指南

今天花了一些时间配置 OpenClaw AI 助手与飞书的集成，过程有些曲折，但最终成功了。记录一下配置过程和遇到的问题，给后来者参考。

## 什么是 OpenClaw？

OpenClaw 是一个 AI 助手框架，可以让你拥有自己的 AI 助手，并集成到各种通讯平台中。它支持多种消息渠道，包括飞书、Telegram、WhatsApp 等。

## 为什么选择飞书？

飞书（Feishu）是字节跳动推出的企业协作平台，集成了即时通讯、日历、文档、视频会议等功能。选择飞书作为 AI 助手的交互界面，主要是因为：

1. **日常使用频繁** - 工作和沟通都在飞书上
2. **消息同步好** - 多设备消息实时同步
3. **API 完善** - 飞书开放平台提供了丰富的 API
4. **机器人支持** - 可以方便地创建和配置机器人

## 配置过程

### 1. 准备工作

首先需要确保以下环境就绪：

- Node.js 环境（推荐 v20+）
- OpenClaw 安装完成
- 飞书开发者账号

### 2. 安装 OpenClaw

```bash
npm install -g openclaw
```

### 3. 配置飞书集成

在 OpenClaw 的配置文件中添加飞书渠道：

```yaml
channels:
  feishu:
    appId: "你的飞书应用 App ID"
    appSecret: "你的飞书应用 App Secret"
    verificationToken: "验证 Token"
    encryptKey: "加密密钥"
```

### 4. 创建飞书应用

1. 登录 [飞书开放平台](https://open.feishu.cn/)
2. 创建新应用
3. 配置应用权限（需要消息、机器人等权限）
4. 获取 App ID 和 App Secret
5. 配置事件订阅和消息接收地址

### 5. 启动 OpenClaw

```bash
openclaw start
```

## 遇到的问题及解决方案

### 问题 1：消息无法接收

**现象**：飞书消息发送后，OpenClaw 没有响应

**原因**：事件订阅 URL 配置错误

**解决**：确保回调地址正确指向 OpenClaw 的 webhook 端点

### 问题 2：权限不足

**现象**：机器人无法发送消息到群聊

**原因**：飞书应用权限配置不完整

**解决**：在飞书开放平台添加必要的权限，如：
- 消息发送权限
- 群组读取权限
- 机器人功能权限

### 问题 3：消息同步延迟

**现象**：手机端能看到消息，电脑端延迟

**原因**：飞书客户端缓存或网络问题

**解决**：
- 刷新飞书客户端
- 检查网络连接
- 重启飞书应用

整个配置过程大约花了几个小时，主要时间花在理解飞书开放平台的权限配置和 OpenClaw 的渠道设置上。一旦配置完成，使用体验非常流畅。

3. 测试时先用个人账号，再扩展到群聊
4. 遇到问题查看 OpenClaw 的日志输出
